Right now I am trying to make a 4'' TFT SPI LCD panel work. There is a lot of information to take in, especially for someone who's not that profficient in electronics. The largest obstacle is figuring out the ins and outs of the SPI interface, although, I am starting to understand how things work. First, set the CS (Chip Select) to low in order to enable the communication between the "Master" (Arduino) and the "Slave" (LCD panel). Then you need to specify whether you'll be sending a command or data using the D/C pin (Data/Command). You have to look up the hex codes for each command and send it via the MOSI pin (Master Out - Slave In). Then start sending the parameters of the command by setting the D/C pin to low. My initial roadblock was figuring out that Arduino has assigned specific pins for the SPI protocol. So when you first try to use the built-in SPI libary make sure you've connected everything up correctly.
